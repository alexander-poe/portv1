Family Page

  • Overview: This photo sharing app was made with two colleagues and myself to create a better way to share photos between families. I was responsible for design, mobile-responsiveness and gallery features.


    Source Live

  • Challenge: The challenge was managing the state of the React app with multiple one to one and one to many relationships being served from the backend. Going into this project we wanted to use as little plugins and packages as possible to be challenged with writing the code from scratch.

  • Tech: My favorite part of this project was the process of finding the best solution to manage the state of different users in relation to their photos, comments, and replies. Using redux and carefully crafted reducers we managed the state of the application flawlessly.

  • Takeaways: Managing the state of the front end application and data structuring can have a wide variety of reasonable approaches. Taking the time to critically plan out the right approach suited for the clients is a valuable thing to do and served us well in the creation of Family Page.

InstaFrench

  • Overview: Do you need to touch up on your french? You are in luck, InstaFrench, a flashcard like approach using a spaced repetition algorithm will get you a step closer to your learning goals one card at a time.

    Source Live

  • Challenge: The challenge of this project was to create a UX that would emulate the experience of learning with flashcards. This project was also the first project where I used Passport, Bearer and Oauth to authenticate a user's profile.

  • Tech: Having the CRUD API interact with the front-end while being authenticated posed some interesting yet predictable challenges. Using react-cookies to send authentication tokens in the header of our fetch calls allowed us to retrieve data securely and privately.

  • Takeaways: UX is at the helm of modern development, eCommerce and of critical importance to create products that work seamlessly with what the user wants. Modeling modern days apps to feel like real world things provides benefit in familiarity and usability.

Recipe Search

  • Overview: Discover recipes, see their ingredients and nutritional facts without needing to open new tabs. Vegan, GF, Vegetarian, Peanut allergies? You're in luck, we use filters to provide suggestions for your needs. This project was created while pair programming, allowing both of us to talk about different approaches to arrive upon best solution.


    Source Live

  • Challenge: Going into this project we wanted to create a UX that provided a lot of value in content with minimal clicks. I had experienced that searching for recipes lead to an over clustered browser full of tabs, we wanted to solve this problem with a one stop shop for finding great recipes.

  • Tech: For this project we used Edamames API and jQuery for presentation and AJAX. JQuery provided a very easy to use framework for writing essential front end operations like conditional component like rendering, handling events and API calls.

  • Takeaways: JQuery can be a powerful solution for simple data display applications and can appropriate for projects of this scope. I find jQuery very easy to work with yet I have also seen jQuery codebases start to look like a bowl of code spaghetti. This is where component based frameworks like Ember.js and React.js shine because they separate concerns architecturally making the app easier to manage as it scales.
