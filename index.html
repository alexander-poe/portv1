Fuzzy Finder

  • Overview: Fuzzy finder is exactly that! It is auto suggestion so you can find what you are looking for even when you don't know exactly what the title is. This was a solo project and a great learning experience into the world of fuzzy finders, which has large degrees of varying complexity.

    Source Live

  • Challenge: This app posed the challenge of how far down the rabbit hole I wanted to go. From approximate string matching, to auto suggestions based on phonetics the rabbit hole got deep quickly.

  • Tech: I ended up using an algorithm that matches the characters of the text with the query in relation to order of the characters and character position. Not to mention, the dataset the algorithim is using is over 1500 entries and the runtime of this problem could sky rocket quickly.

  • Takeaways: In this project I learned a lot about how to work with more bigger data and how to create a system that could handle such. The performance was clearly noticeable with different solutions.

InstaFrench

  • Overview: Need to touch up on your french? You are in luck, instaFrench, a flashcard like approach using a spaced repetition algorithm will get you a step closer to your learning goals.

    Source Live

  • Challenge: The challenge of this project was to create a simple UI that would represent a flashcard-like approach to learning. We ran into a challenge with authentication that I was able to debug by installing react-cookies and sending cookies in the header of the action. I will further elaborate on the next slide.

  • Tech: Essentially with passport's google authentication a user signs in with their google information and an authentication token is stored in the cookies. Bearer, used for route authentication requires the authentication token in cookies for access. When the cookie was not included in the request to the server, Bearer did its job and denied access.

  • Takeaways: This project was a fun with a lower level of complexity and moving parts compared to the projects I am working on now. I'd like to explore making this entire project dynamic in a way that a user could customize the theme of the project and the questions that are asked. It could be an all around much more useful tool with those capabilities.

Color Helper

  • Overview: Want to create a color scheme for your app in a more efficent intuitive way? This is the app for you, with Color Helper you can adjust the colors of basic elements to establish your main, secondary, text, and highlight color.

    Source Live

  • Challenge: The challenge of this project to make an easier way to choose the color scheme of your app without the use of dev tools or frequent reloads. Futhermore, to do this in way that is dynamic enough that you could reuse the same components for views in different screen sizes.

  • Tech: This app uses styled-components which is a react package for writing inline styles without creating seperate style objects. Basically, each component displayed in the Iphone mockup will change based on the result of the control input. This way you can not only easily try out multiple colors but can take the #hex codes immediately and use them in your app.

  • Takeaways: Where to start, I have quite a few idea with this technology. In the same way that I altered the colors, I could alter any property of the css. Now I would have to do this in a way that would maintain scalability of course. I'd like to have multiple color control options, for phones, tablets and desktops to accomodate whatever the user needs.
